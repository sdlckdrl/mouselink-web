Technical & Security

How it connects and which permissions it uses — laid out separately.

This page isn't the everyday how-to. It explains how the connection works and why each permission is used. For step-by-step usage, see the User Guide.

Local first Permissions Data handling Network Support scope Update log

Local First

It works only between your own paired devices.

Input, files, and clipboard travel between your PC and Android (or another PC) on the same network. Nothing is uploaded to an account through an outside server. That's why the one condition for connecting is that both devices sit under the same Wi-Fi (the same Wi-Fi router) — or are connected directly over Bluetooth.

Permissions

Only what a feature actually needs.

Nothing is requested all at once — you turn each one on yourself on Android, right when that feature starts.

Required

Accessibility Service

To control Android with your PC's mouse and keyboard and to handle remote input and screen-related actions, the Accessibility Service is required. OneMouse's core features depend on it.

When starting remote control

Screen capture approval

Approved through an Android system prompt when you open the mobile remote control window. Some content, such as secure screens, may not appear.

When connecting

Nearby Wi-Fi devices

May be requested on Android 13 and later when finding your PC. It's asked the moment you search, not when the app launches.

When the service starts

Notifications

Used to show the background connection status. Requested only when needed.

When using brightness

Modify system settings

Needed once the first time you adjust the mobile screen brightness with Alt+Shift+↑/↓.

Recommended

Battery optimization exception

Recommended for stability if the connection drops after the screen turns off. It isn't required.

Data Handling

Input, files, and clipboard — only within the scope you choose.

  • No remote account is created — it works through pairing, with no sign-in.
  • Files you send aren't stored on a server. They go straight to the device you pick.
  • Accessibility data is never used for advertising or screen collection. It's used only for the stated features, such as passing input and remote control.
  • Clipboard and file transfers happen only when you choose a target and start them yourself.

Network

Connection quality depends on your local network conditions.

Recommended setup

  • The same Wi-Fi on a personal Wi-Fi router (2.4 or 5 GHz, either is fine)
  • A network that allows device-to-device communication
  • OneMouse allowed inbound in the Windows firewall

Limited setup

  • Client isolation on guest or corporate Wi-Fi
  • Inbound blocked on a network marked as "Public"
  • In these cases, use a direct Bluetooth connection or a personal hotspot

OneMouse doesn't go through an internet relay server, so the two devices need to be able to find each other directly.

Support Scope

The public guide is based on Wi-Fi/LAN and Bluetooth connections.

  • Supported: Windows 10/11 + Android, over Wi-Fi/LAN on the same network and direct Bluetooth connections.
  • Not included: connecting over USB/ADB is outside the official support scope.
  • PC-to-PC works with main / sub roles and supports two-way file transfer.
