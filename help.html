Help

권한부터 연결 문제까지, 자주 막히는 지점을 모아 두었습니다.

설치 직후 권한이 헷갈리거나, 검색은 되는데 제어가 안 되는 경우를 빠르게 점검할 수 있도록 도움말 항목을 모듈처럼 나눠 두었습니다.

Permissions

필수 권한 안내

  • 입력 제어를 받으려면 Android 쪽 접근성 또는 입력 관련 권한이 필요합니다.
  • 파일을 받으려면 저장소 또는 파일 접근 권한이 필요한 환경이 있을 수 있습니다.
  • 권한이 꺼져 있으면 검색은 되어도 실제 제어나 전송이 동작하지 않을 수 있습니다.

Connection

연결이 보이지 않을 때

  • PC와 Android가 서로 다른 게스트 Wi-Fi나 분리된 네트워크에 있지 않은지 확인합니다.
  • 회사 또는 공용 Wi-Fi에서는 로컬 브로드캐스트가 제한될 수 있습니다.
  • Windows 방화벽이나 백신이 앱의 네트워크 통신을 막고 있지 않은지 확인합니다.
  • Android 앱이 백그라운드에서 강제 종료되었다면 다시 열어 대기 상태로 둡니다.

Layout

마우스 진입 위치가 어색할 때

  • 모바일 위치가 실제 모니터 외곽과 같은 방향인지 다시 맞춰 봅니다.
  • 여러 기기를 붙인 경우엔 각 모바일이 서로 겹치지 않게 배치하는 편이 좋습니다.
  • 듀얼 모니터에서는 어느 모니터의 외곽으로 진입할지 기준을 먼저 정해 두면 편합니다.

Transfer

파일 전송 팁

  • 파일은 현재 선택된 기기로 전송되므로 PC에서 어떤 기기가 선택되었는지 먼저 확인합니다.
  • 대용량 파일은 네트워크 상태에 따라 시간이 걸릴 수 있으니 진행 상태를 함께 보는 편이 좋습니다.
  • 테스트 APK, 스크린샷, 문서 파일처럼 자주 쓰는 자료 전송에 특히 잘 맞습니다.

Troubleshooting

문제 해결 순서

대부분은 아래 순서대로 점검하면 빠르게 원인을 좁힐 수 있습니다.

1
네트워크 확인

두 기기가 같은 공유기 아래에 있는지, 로컬 통신이 막히지 않는지 먼저 확인합니다.

2
앱 상태 확인

PC 앱과 Android 앱이 모두 실행 중이고, 연결 대기 또는 연결 상태인지 확인합니다.

3
권한 확인

접근성, 입력, 파일 관련 권한이 꺼져 있지 않은지 다시 점검합니다.

4
다시 연결

필요하면 페어링을 다시 시도하거나 앱을 재실행해 연결 상태를 새로 잡습니다.

FAQ

자주 묻는 질문

사용 중 자주 나오는 질문들을 빠르게 훑어볼 수 있게 정리했습니다.

중간 서버를 거치나요?
기본 구조는 같은 로컬 네트워크 안에서 PC와 Android가 직접 통신하는 방식입니다.
왜 6자리 페어링 코드가 필요한가요?
처음 연결 시 같은 기기끼리 연결되는지 확인하기 위한 절차로, 우발적인 연결을 줄여 줍니다.
여러 Android를 동시에 붙일 수 있나요?
가능합니다. PC에서는 여러 기기를 보고 배치할 수 있고, 배치된 위치에 따라 제어가 넘어갑니다.
모바일에서는 왜 다른 기기가 보이기만 하나요?
모바일 배치 화면에서는 전체 배치를 이해할 수 있도록 다른 기기도 보이지만, 직접 조정은 자신의 기기만 가능합니다.
파일 전송은 어느 기기로 가나요?
현재 선택된 기기가 파일 전송 대상입니다. PC 기기 관리에서 선택 상태를 확인하세요.
키보드 입력이 이상하게 들어가면 어떻게 하나요?
Android 입력기 상태, 언어 전환, 접근성 권한을 먼저 확인하고, 앱을 다시 연결하면 정상화되는 경우가 많습니다.
