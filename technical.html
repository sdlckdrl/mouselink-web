연결 구조와 권한을 따로 정리했습니다.

이 페이지는 일반 사용법이 아니라 연결이 어떻게 되는지와 권한을 왜 쓰는지를 설명합니다. 사용 순서는 사용 가이드를 보세요.

로컬 우선 권한 사용 데이터 처리 네트워크 조건 지원 범위 업데이트 내역

Local First

페어링한 내 기기 사이에서만 동작합니다.

입력·파일·클립보드는 같은 네트워크의 내 PC와 Android(또는 다른 PC) 사이에서 오갑니다. 외부 서버를 거쳐 계정에 올리는 구조가 아닙니다. 그래서 연결의 전제 조건은 두 기기가 같은 Wi-Fi(같은 공유기) 아래에 있는 것(또는 블루투스로 직접 연결)입니다.

Permissions

권한은 기능 제공에 필요한 범위로만.

한꺼번에 요구하지 않고, 그 기능을 시작할 때 Android에서 사용자가 직접 켭니다.

필수

접근성 서비스

PC의 마우스·키보드로 Android를 제어하고, 원격 입력·화면 관련 동작을 처리하려면 접근성 서비스가 필요합니다. OneMouse의 핵심 기능이 여기에 의존합니다.

원격 제어 시작 시

화면 캡처 승인

모바일 원격 제어 창을 열 때 Android 시스템 팝업으로 승인합니다. 보안 화면 등 일부는 표시되지 않을 수 있습니다.

연결 시

주변 Wi-Fi 기기

Android 13 이상에서 PC를 찾을 때 요청될 수 있습니다. 앱 실행 시가 아니라 검색하는 순간에만 물어봅니다.

서비스 시작 시

알림

백그라운드 연결 상태를 표시하는 데 씁니다. 필요할 때만 요청됩니다.

밝기 사용 시

시스템 설정 변경

Alt+Shift+↑/↓로 모바일 화면 밝기를 조절할 때 처음 한 번 필요합니다.

권장

배터리 제한 예외

화면이 꺼진 뒤 연결이 끊긴다면 안정성을 위해 권장합니다. 필수는 아닙니다.

Data Handling

입력·파일·클립보드는 사용자가 선택한 범위에서.

  • 원격 계정을 만들지 않습니다 — 로그인 없이 페어링으로 동작합니다.
  • 전송한 파일을 서버에 저장하지 않습니다. 선택한 기기로 직접 보냅니다.
  • 접근성 데이터를 광고나 화면 수집 용도로 사용하지 않습니다. 입력 전달·원격 제어 등 명시된 기능에만 씁니다.
  • 클립보드·파일 전송은 사용자가 대상을 선택하고 실행할 때만 일어납니다.

Network

연결 품질은 로컬 네트워크 조건의 영향을 받습니다.

권장 환경

  • 개인 공유기의 같은 Wi-Fi(2.4/5GHz 무관)
  • 기기 간 통신이 허용된 네트워크
  • Windows 방화벽에서 OneMouse 인바운드 허용

제약 환경

  • 게스트·회사 Wi-Fi의 기기 간 격리(client isolation)
  • ‘공용’으로 잡힌 네트워크의 인바운드 차단
  • 이럴 땐 블루투스 직접 연결 또는 개인 핫스팟 이용

OneMouse는 인터넷 릴레이 서버를 거치지 않으므로, 두 기기가 서로를 직접 찾을 수 있는 환경이어야 합니다.

Support Scope

공개 가이드는 Wi-Fi/LAN과 블루투스 연결을 기준으로 합니다.

  • 지원: Windows 10/11 + Android, 같은 네트워크의 Wi-Fi/LAN 및 블루투스 직접 연결.
  • 제외: USB/ADB를 통한 연결은 공식 지원 범위가 아닙니다.
  • PC↔PC는 메인/서브 역할로 동작하며 양방향 파일 전송을 지원합니다.
