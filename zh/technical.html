技术与安全

连接结构与权限，我们单独做了整理。

本页不讲一般用法，而是说明连接是怎么建立的以及为什么要用这些权限。使用步骤请查看使用指南。

Local First

只在你配对过的设备之间运行。

输入、文件和剪贴板，只在同一网络下你自己的 PC 与 Android（或另一台 PC）之间往来。它不会经过外部服务器上传到某个账号。因此连接的前提是两台设备处在同一 Wi-Fi（同一 Wi-Fi 路由器）之下（或通过 Bluetooth 直接连接）。

Permissions

权限只用于提供功能所需的范围。

不会一次性全部索取，而是在你开始使用某项功能时，由你在 Android 上亲自开启。

必需

无障碍服务

要用 PC 的鼠标和键盘控制 Android，并处理远程输入及屏幕相关操作，就需要无障碍服务。OneMouse 的核心功能都依赖于它。

Data Handling

输入、文件和剪贴板，都在你选定的范围内。

  • 不创建远程账号 — 无需登录，通过配对即可运行。
  • 不会把你发送的文件保存到服务器。而是直接发送到你选择的设备。
  • 不会把无障碍数据用于广告或屏幕采集用途。只用于输入传递、远程控制等明示的功能。
  • 剪贴板和文件传输，只在你选定对象并执行时才会发生。

Network

连接质量会受本地网络条件的影响。

推荐环境

  • 个人 Wi-Fi 路由器下的同一 Wi-Fi（2.4/5GHz 均可）
  • 允许设备间通信的网络
  • 在 Windows 防火墙中允许 OneMouse 的入站连接

受限环境

  • 访客或公司 Wi-Fi 中的客户端隔离（client isolation）
  • 被识别为“公用”的网络会阻止入站连接
  • 此时请改用 Bluetooth 直接连接或个人热点

OneMouse 不经过互联网中继服务器，因此需要两台设备能够彼此直接找到的环境。

Support Scope

公开指南以 Wi-Fi/LAN 和 Bluetooth 连接为准。

  • 支持：Windows 10/11 + Android，同一网络下的 Wi-Fi/LAN 以及 Bluetooth 直接连接。
  • 不含：通过 USB/ADB 的连接不在官方支持范围内。
  • PC↔PC 以主/从角色运行，并支持双向文件传输。
